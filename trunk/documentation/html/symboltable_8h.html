Detailed Description

The Interface/Specification for our SymbolTable (symboltable).

The Symbol Table (symboltable) object is a core object used by the Scanner and Parser and the majority of the compiler altogether.

We impliment the symbol table as a Map of a Map of a Map. The reason behind this is that it allows any given component of the table to have multiple values if needed. Additionally, the nested Map structure allows for fast searching and direct access as well.

Traditionally, a Symbol Table will be constructed for each "block" of nesting we enter.

It should be noted that "lexeme" is used frequently but in most cases the given function / table entry may not be a traditional lexeme at all. We provide functions to find a symbol table entry with any 2 given entry values.
